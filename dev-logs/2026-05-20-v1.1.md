# 开发日志 — 2026-05-20 v1.1

## Phase 1：任务书导入基础设施

### 文件分析结果
- 21个真实 Excel 文件，8个 `.xls`（旧格式）+ 13个 `.xlsx`
- 两类格式：
  - **核心业务梳理表**（14个）：单 sheet，10列，表头含换行符（如 `指标\n来源`）
  - **综合考核指标解构情况表**（7个）：多 sheet，10-15列，每 sheet 对应一个评价部门
- 核心发现：表头包含换行符导致精确匹配失败；单位名称可能不完全一致

### 代码改动
| 文件 | 改动 |
|------|------|
| `backend/utils/xlsx_handler.py` | 新增 `read_xls_sheets`、`read_xlsx_sheets`、`read_workbook`（自动识别.xls/.xlsx）、`fuzzy_match_headers`（关键词模糊匹配+去重）、`HEADER_KEYWORDS` 映射表 |
| `backend/routes/task.py` | 重写导入逻辑：`read_workbook` 替代直接 openpyxl → 精确匹配+模糊匹配双模式 → `_import_unified_rows` 统一导入；新增 `_find_unit`/`_find_dim` 模糊查找（精确→包含→去前缀降级）；新增 `_normalize_period`（周期标准化）；`SKIP_SHEET_KEYWORDS` 跳过非数据工作表 |
| `requirements.txt` | 新增 `xlrd>=2.0` |
